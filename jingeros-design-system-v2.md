## 0. Connie 的修订说明

这份文档在 Gemini / ChatGPT 所产出的三份设计文档基础之上重写。以下是我认为需要调整的关键点，以及我的理由：

**CRT 预览帧率：3fps → 5fps。** 3fps 的间隔（每帧 333ms）在实际观感上会被用户理解为"卡了"而不是"老旧设备"。5fps（200ms/帧）仍然足够传递录像带质感，但不会触发"这网站有 bug"的本能反应。配合扫描线和噪点叠加，5fps 已经完全足够还原 VHS 预览的味道。

**Glitch 过渡总时长：1.3s → 1.8s。** 这是整个体验中最壮观的一刻——信号从模拟硬件中逃逸出来变成数字界面。1.3s 太快了，用户来不及注册发生了什么。1.8s 刚好够让每个阶段的视觉变化都被看见、被感受到，但不至于让人等得不耐烦。一个 Awwwards 级的过渡不是快，是节奏感。

**弧形排列角度：180° → 135°。** 完整半圆会让两端磁带太接近水平线，视觉上像"排开"而不是"围绕"。135° 的弧度让整个排列呈现轻微的拥抱姿态——像一群人围着篝火坐，而不是站成一排。CRT 在最底部居中。

**声音设计：默认静音。** 我见过太多优秀的网站因为自动播放声音被用户秒关。声音应该是"发现"而不是"强塞"。第一次 hover 到磁带时，在 CRT 屏幕角落温和提示一次"🔇 Click to enable sound"，之后所有声音系统激活。

**移动端策略：完全不同的方案。** VHS 物理交互在移动端不可行。移动端使用卡片式布局 + 保留 VHS 标签视觉元素 + 简化过渡动画。不降级，而是针对触屏重新设计。

**返回流程增加一个"确认帧"。** 浏览器坍缩后，CRT 屏幕先显示当前项目的最后一帧静止画面 0.3s，然后再开始关机序列。这 0.3s 让用户确认"我确实是从那个项目回来的"，心理上完成了闭环。

## 1. 核心理念

### 这个网站是什么

这不是一个作品集网站。这是一台被遗忘的个人电脑，里面保存着多年的创作记忆。

访问者不应该觉得自己在"浏览项目列表"。他们应该觉得自己发现了一台还在运行的旧电脑，打开了一个尘封的抽屉，从里面翻出了几盘磁带，然后一盘一盘地插进电视机里看完了所有录像。

### 叙事内核

桌面代表现在。档案室代表过去。每个项目是一段等待被重建的记忆。

这个隐喻和 Jinger 的个人 tagline 形成呼应："From words to code. Still translating." 翻译本身就是一种信号重建——从一种介质转化到另一种介质，在损耗和噪声中保留意义。

### 体验的三个阶段

**阶段一：好奇** — 桌面看起来很熟悉。访问者认出了操作系统界面，开始探索。一切看起来正常。

**阶段二：发现** — 点击 Projects.app 不是打开另一个窗口，而是整个操作系统暗下来，一台 CRT 电视从桌面深处升起。访问者意识到这个网站还有另一层。

**阶段三：沉浸** — 选择一盘磁带、插入电视、经历信号重建、进入一个全新的数字世界。每个项目都有自己的界面语言。每个项目都像另一个产品而不是另一个案例研究。到这一步，访问者已经忘记自己在看作品集了。

### 3.3 材质语言

网站中每个可见物体都属于一种物理材质类别。渲染语言必须一致地传达这些材质属性。

**Material A — CRT 塑料外壳**
略粗糙的注塑成型塑料。细微表面纹理。微小划痕。圆角。柔和漫反射。永远不要光滑如镜。大范围柔和高光，极弱镜面反射，接缝处有微弱环境光遮蔽。

**Material B — CRT 玻璃屏幕**
弯曲。厚重。沉重。略带绿色色调。边缘畸变。可见反射、水平扫描线、轻微泛光、偶尔的灰尘颗粒、轻度桶形畸变。没有完美平坦的反射。

**Material C — VHS 卡带外壳**
半哑光黑色塑料。微小划痕。注塑痕迹。机械螺丝。小的制造瑕疵。应立即传达物理重量感。

**Material D — VHS 透明视窗**
略烟熏的亚克力。非常柔和的反射。内部磁带卷轴可见。微观灰尘。微弱的边缘折射。

**Material E — 纸质标签**
略微泛黄的纸张。手写笔记。小皱纹。胶带残留。每个项目应有独特的标签设计。标签应看起来是手动贴上的而不是数字生成的。

**Material F — Chrome 浏览器框架**
完美平坦。现代。极简。干净。无老化。无划痕。无复古处理。浏览器有意与物理档案形成对比。

### 3.4 光照系统

#### Desktop 光照
大面积环境光。柔和阴影。极少对比度。安全的环境感。

#### Archive 光照
单一主光源：CRT 屏幕。
次要光源：微小的 LED 指示灯、微弱的环境反弹照明。
角落保持黑暗。物体逐渐消失在阴影中。访问者的眼睛自然被引向电视机。

### 3.5 深度与层次

网站应始终传达多个空间层次：

| 层级 | Desktop | Archive |
|------|---------|---------|
| 前景 | 光标 | 磁带（hover态） |
| 交互层 | 活跃窗口 | 磁带（静止态） |
| 主要对象 | 桌面图标 | CRT 电视 |
| 背景 | 壁纸 | 暗色档案室 |
| 大气层 | 无 | 漂浮灰尘颗粒、微弱烟雾 |

### 3.6 表面老化规则

所有物理物体都不应该看起来像全新出厂的。

允许的瑕疵：发丝划痕、灰尘、轻微指纹、纸张变色、微小凹痕、塑料磨损、螺丝氧化、非常微弱的边缘褪色。

这些细节应保持几乎不可见，直到仔细观察才能发现。它们的目的是潜意识层面的真实感而不是装饰。

档案室保存记忆。它被精心维护。它是旧的，但被尊重的。

## 4. 项目展示流程：VHS Archive Experience

### 4.0 流程总览

```
Desktop → [点击 Projects.app]
  → Archive 场景升起（CRT + 5盘磁带）
    → Hover 磁带：磁力吸附 + CRT 预览
    → 点击磁带：贝塞尔曲线飞向 CRT → 插入
      → Glitch 信号重建（1.8s）
        → 浏览器从信号中展开
          → 项目独立网站
            → 点击返回：浏览器坍缩 → CRT 关机 → 磁带弹出 → 回到 Archive
```

### 4.1 触发：Desktop → Archive

**触发方式：** 双击桌面上的 `Projects.app` 图标，或从 Dock 点击 Projects 图标，或在 Terminal 输入 `/projects`。

**过渡序列（总时长约 1.2s）：**

1. **T+0ms** — 桌面背景开始暗化（opacity 从 1 降到 0.35），使用 `ease-out` 曲线，持续 600ms
2. **T+100ms** — 所有其他桌面图标施加 6-8px 高斯模糊 + opacity 降至 0.4，持续 500ms
3. **T+200ms** — Dock 停止所有动画（bounce、tooltip），opacity 降至 0.5
4. **T+300ms** — CRT 电视从桌面中央偏下位置开始升起
   - 起始位置：viewport 底部下方 100%
   - 终止位置：viewport 垂直居中偏下 10%（给磁带弧形留空间）
   - 运动曲线：`cubic-bezier(0.16, 1, 0.3, 1)` — 快速出发，缓慢到达
   - 持续时间：800ms
   - 附带微弱的上升弹跳（overshoot 约 8px，回弹 200ms）
5. **T+500ms** — CRT 屏幕开始亮起
   - 中央水平亮线先出现（宽度从 0 扩展到 100%，高度 2px）
   - 亮线纵向扩展填充整个屏幕（200ms）
   - 扫描线纹理淡入
   - 屏幕显示待机文字："-- VHS ARCHIVE -- SELECT A TAPE"
   - 字体：`'Press Start 2P'`，颜色：`--archive-green-glow`
6. **T+800ms** — 5盘磁带从 CRT 两侧飞入，沿弧线到达各自位置
   - 每盘间隔 80ms 依次出现（像扑克牌被发出）
   - 运动曲线：`cubic-bezier(0.34, 1.56, 0.64, 1)` — 带轻微弹性
   - 最终排列：135° 弧形，CRT 在弧底部中央

**Menu Bar 行为：** 保持可见，但文字变为 "VHS Archive" 模式的显示。

### 4.2 VHS Archive 场景

#### 空间布局

CRT 电视占据视觉中心偏下位置。5盘 VHS 磁带沿 135° 弧形排列在 CRT 上方和两侧。

```
        Tape 01 (Remoire)
   Tape 02          Tape 03
  (CareerForge)   (French Quest)
   Tape 04          Tape 05
  (Chatbox)       (Generator)
        [CRT Television]
```

弧形半径约为 CRT 宽度的 1.8 倍。磁带之间保持足够间距避免视觉拥挤。

#### CRT 电视设计规格

- 外形参考：Sony Trinitron 14 寸（圆角矩形外壳 + 弧面屏幕）
- 屏幕区域纵横比：4:3
- 渲染尺寸：约 viewport 宽度的 35-40%（桌面端）
- 屏幕默认状态：深绿色微光 + 缓慢移动的扫描线 + 待机文字
- 外壳上可见：品牌标识区域（可放 "JingerOS" 标识）、电源指示灯（红→绿切换）、VHS 插槽（水平条纹状开口）
- 有轻微的 CSS 阴影模拟重量感

#### 环境氛围动画（无交互时）

即使没有用户操作，Archive 场景也不应该感觉冻结：

- CRT 发出几乎不可察觉的呼吸光（每 4-6 秒一次微弱亮度波动）
- 微小灰尘颗粒在电视前方缓慢漂移（6-10个粒子，速度 0.3-0.8px/frame，opacity 0.15-0.3）
- VHS 磁带保持近乎静止，仅有微观级别的闲置运动（类似细线悬挂的物体在安静房间里的微动，幅度 <1px）
- 扫描线以不规则间隔缓慢穿过 CRT 表面
- 偶尔的模拟干扰产生单条不稳定的水平线，随即消失

这些环境动画应微妙到截图看起来几乎是静态的。只有观察页面几秒钟后，访问者才会注意到一切都在静静地活着。

### 4.3 磁带交互

#### Hover 行为

**磁力吸附场**
- 吸引半径：磁带边界外 80-120px
- 光标进入半径后，磁带开始向光标方向微微倾斜
- 越靠近中心，吸引越强
- 磁带永远不要直接跳向光标——应该略微滞后，仿佛重量产生了阻力
- 光标离开吸引场后，磁带用临界阻尼弹簧运动（critically damped spring）缓慢回到静止位置，持续约 400ms

**浮起与倾斜**
- 磁带从 Archive 平面浮起约 12px
- 根据光标位置旋转 2-4°（透视变换的原点在磁带自身而非摄像机）
- 下方阴影增强（blur 从 4px 增加到 12px，spread 增大，颜色加深）

**材质反馈**
- 透明视窗上出现柔和的亚克力高光，随光标位置动态移动
- 高光应像光线穿过烟熏塑料，而不是通用的光泽效果
- 内部磁带卷轴变得略微更可见
- 纸质标签通过阴影深度变化与塑料外壳产生微弱分离感

**CRT 预览（同步发生）**
- Hover 到某盘磁带时，CRT 屏幕立即开始播放该项目的低保真预览
- 帧率：5fps（每 200ms 切换一帧）
- 叠加效果：水平扫描线（2px 间距）+ 随机噪点（opacity 0.08）+ 轻微暗角（radial-gradient 黑色 vignette）+ 微弱的色彩偏移（RGB通道水平偏移 1px）
- 预览内容：项目首页的截图（预先准备好的低分辨率版本）
- Hover 切换到另一盘磁带时，先闪一帧白噪声（50ms），然后切换到新预览
- 无 Hover 时回到待机文字

### 4.4 磁带选择（点击）

点击磁带是整个体验从"网站"变成"机器"的分界点。

**T+0ms — 锁定**
- 被选中的磁带立即成为唯一活跃对象
- 其他四盘磁带 opacity 降至 0.25，scale 缩小至 0.95，持续 300ms
- 桌面进一步暗化（overlay opacity 从 0.35 升到 0.6）
- 禁止所有其他交互。系统进入"播放模式"

**T+200ms — 飞行**
- 磁带沿精心设计的贝塞尔曲线飞向 CRT 插槽
- 直线运动 = 计算感。曲线运动 = 物理感
- 飞行过程中磁带平滑旋转 90°，使插入边对准 VHS 插槽方向
- 速度在接近电视时减慢（先快后慢的 ease-out 曲线）
- 飞行总时长：500-600ms

**T+750ms — 停顿**
- 磁带到达插槽前方约 5px 处，短暂停顿
- 持续 250ms
- 静默。等待。期待。

**T+1000ms — 插入**
- 磁带向前移动最后几厘米进入插槽
- 包含微弱的机械阻力感（非线性运动：慢-快-停）
- 最后一刻的"咔嗒"锁定：磁带到位后电视机抖动 1-2px（持续 80ms）
- 这个微小的振动传达了内部发生了机械动作

**音效（如果声音已开启）：**
- 飞行中：磁带悬浮的轻微气流声
- 插入时：机械卡扣声 "咔嗒"
- 锁定后：电子嗡鸣开始升高

### 4.5 信号重建 — The Big Bang Transition

磁带锁定后，电视的行为完全像一台真实的模拟设备。

**电力先于图像。光先于信号。信号先于内容。**

总时长：1.8s，分为 5 个阶段。

#### Phase 1 — 点火 Ignition（0-300ms）
- CRT 屏幕中央出现一条水平亮线，从中心向两端快速扫描
- 明亮的电子泛光短暂照亮周围的档案室
- 屏幕从纯黑色爆发为不稳定的白色闪烁
- 背景可见微弱的 CRT 电流声

#### Phase 2 — 水平撕裂 H.Tearing（300-600ms）
- 图像尝试稳定但失败
- 水平同步丢失——画面出现上下撕裂
- 垂直同步挣扎——画面抖动
- 扫描线变得非常明显（宽度增加到 3-4px）

#### Phase 3 — RGB 分离 RGB Split（600-900ms）
- RGB 三个通道分别水平偏移 3-6px
- 红色通道向左偏
- 蓝色通道向右偏
- 绿色通道保持中央但垂直抖动
- 此阶段，未来项目的碎片内容开始在干扰中隐约可见
  - 也许是一个标题
  - 也许是一种颜色
  - 也许是一个界面元素的轮廓
  - 不够清晰到可以阅读，但足够让人产生期待

#### Phase 4 — 雪花噪声 Snow Storm（900-1200ms）
- 全屏白噪声静电
- 间歇性地闪过项目画面（每次 50-80ms）
- 噪声密度逐渐降低——信号逐渐获胜
- 亮度波动剧烈

#### Phase 5 — 信号逃逸 Screen Expand（1200-1800ms）
- 画面突然稳定 100ms
- 然后 CRT 画面开始突破电视的物理边界
- 不是浏览器淡入覆盖电视——是画面从信号中生长出来
- CRT 边缘溶解，画面向四周扩展
- 扩展速度先慢后快（ease-in 曲线）
- 最后 200ms 快速填满整个 viewport
- 所有 CRT 效果（扫描线、噪点、色彩偏移）在扩展过程中逐渐消退
- 当画面完全填满 viewport 时，它已经变成了干净的数字画面

**音效序列（如果声音已开启）：**
- Phase 1: CRT 电流声 + 噼啪声
- Phase 2-3: 模拟信号噪声
- Phase 4: 白噪声静电
- Phase 5: 电子嗡鸣渐弱 → 静默

### 4.6 浏览器展开

浏览器只在信号完全稳定后出现。它的运动与前面的模拟序列形成刻意的对比。

**浏览器的运动感受应该是：数学般精确。**

#### 浏览器框架（Chrome Shell）设计

参考早期 Chrome / Safari 浏览器，但不完全复制任何一个。

├─────────────────────────────────────────────────┤
│ ABOUT · PROCESS · GALLERY · IMPACT              │

- 红黄绿三个控制按钮（左上角）
- 地址栏显示 `jingeros://projects/[项目名]`
- 导航栏：ABOUT · PROCESS · GALLERY · IMPACT（视项目而定）
- 框架颜色：纯白 + 浅灰边框 + 铬银控件

#### 展开序列

1. 浏览器框架先出现（顶部 chrome 栏从上方滑入，200ms）
2. 地址栏文字逐字打出（typewriter 效果，150ms）
3. 红黄绿按钮依次亮起（每个 50ms 间隔）
4. 导航控件变为可见
5. 项目网站内容从下方淡入上升（fade-up，300ms）
6. 浏览器框架永远不弹跳、不抖动、不 glitch — 所有不稳定属于它之前的模拟世界

### 4.7 返回流程 — Power Off

离开项目应该与进入项目有不同的情绪。进入代表发现。返回代表保存。

**触发：** 点击浏览器左上角的红色关闭按钮，或浏览器导航栏中的"← Back to Archive"。

#### 返回序列

**T+0ms — 浏览器坍缩**
- 项目内容先消失（fade-down，200ms）
- 浏览器框架向 CRT 电视方向收缩
- 收缩运动使用 `ease-in` 曲线（先慢后快）
- 持续 400ms

**T+400ms — 确认帧**
- 浏览器完全收缩回 CRT 屏幕内
- CRT 短暂显示项目的最后一帧静止画面（0.3s）
- 画面带有轻微的扫描线和 CRT 质感
- 这 0.3s 让用户确认"我从那个项目回来了"

**T+700ms — CRT 关机序列**
- 画面亮度迅速压缩为屏幕中央的一条细水平白线
- 白线从两端向中心收缩
- 最后只剩中心一个明亮的白色光点
- 短暂静默
- 光点消失
- 黑暗。

**T+1200ms — 机械声**
- 一声柔和的机械"咔嗒"
- VHS 磁带从插槽中被弹出几厘米

**T+1400ms — 磁带归位**
- 弹出的磁带沿着与插入时相同的贝塞尔曲线轨迹反向飞回原始位置
- 其他四盘磁带 opacity 恢复到 1，scale 恢复 1
- 桌面亮度恢复到 Archive 模式

**T+2000ms — Archive 恢复**
- 环境粒子继续漂移
- CRT 回到待机状态
- 所有磁带可再次交互
- 用户可以选择另一段记忆

### 4.8 退出 Archive 回到 Desktop

从 Archive 回到 Desktop 的方式：
- 按 ESC 键
- 点击 Menu Bar 上的返回控件
- 在 Archive 空白区域点击

退出序列是进入序列的反向：
1. 磁带向四周散开消失（反向飞入动画）
2. CRT 关机（快速版本，不需要完整关机序列）
3. CRT 向下沉入桌面并消失
4. 桌面模糊和暗化效果反转
5. Dock 恢复动画
6. 回到正常 Desktop 状态

## 5. 动效语言

### 全局原则

动画永远不是装饰。动画解释物理。

访问者应该理解：
- 一个东西有多重
- 它是什么材质做的
- 它如何移动
- 它为什么停下来

每个运动应该回答一个物理问题：
- Hover 揭示重量
- 插入揭示机械结构
- Glitch 揭示信号重建过程
- 浏览器扩展揭示数字重建
- 关机揭示播放结束

### 运动层级

大部分时刻保持安静，少数时刻成为难忘的高潮。像一首音乐。

**高强度动画（整个 Archive 体验中只出现 4 次）：**
1. CRT 从桌面升起
2. VHS 磁带被插入电视
3. CRT 信号爆发为浏览器
4. 电视关机并返回 Archive

**低强度动画（持续进行，大多数访问者只在潜意识中注意到）：**
- Hover 交互
- 光标追踪
- 反射变化
- 扫描线
- 漂浮粒子
- 阴影变化

### 缓动函数库

| 用途 | 函数 | 备注 |
|------|------|------|
| UI 元素出现 | `cubic-bezier(0.16, 1, 0.3, 1)` | 快出慢停 |
| 物理物体移动 | `cubic-bezier(0.25, 1, 0.5, 1)` | 带惯性 |
| 弹性回弹 | `cubic-bezier(0.34, 1.56, 0.64, 1)` | 轻微过冲 |
| 磁带归位 | critically damped spring | k=300, damping=25 |
| 浏览器坍缩 | `cubic-bezier(0.55, 0, 1, 0.45)` | ease-in，加速消失 |
| CRT 关机 | `cubic-bezier(0.7, 0, 0.84, 0)` | 急促压缩 |

## 8. 响应式策略

### 桌面端（>1024px）
完整 VHS Archive 体验。所有物理交互、3D 效果、声音设计。

### 平板端（768-1024px）
- CRT 和磁带尺寸按比例缩小
- 磁带排列改为两行式而非弧形
- 保留 CRT 预览功能
- 简化粒子效果数量
- Glitch 过渡保留但减少阶段数（3 阶段）

### 移动端（<768px）
完全不同的方案。不降级，而是重新设计：

- 磁带变为竖向列表
- 每盘磁带显示为一张带 VHS 标签风格的卡片
- 卡片上保留：磁带编号、项目名、标签纸质感
- 点击卡片：简化的过渡动画（扫描线 overlay + fade）→ 进入项目
- 不使用 CRT 电视模型
- 不使用贝塞尔曲线飞行
- 返回使用标准手势（左滑或顶部返回按钮）

## 9. 性能与技术约束

### 资源预加载策略

- Archive 场景的所有资源（CRT 模型、磁带贴图、预览帧）在 Desktop 状态下就开始预加载
- 使用 `<link rel="preload">` 预加载关键动画资源
- 每个项目的预览帧（5fps 动画用的图片）：每个项目 5-8 帧，单帧 < 50KB
- Glitch 过渡使用 CSS filter 和 canvas 而不是预渲染视频

### 动画性能

- 所有位移和旋转使用 `transform`（GPU 加速）
- 避免在动画中改变 `width/height/top/left`
- 粒子效果使用 `<canvas>` 或 CSS `will-change`
- Glitch 效果使用 WebGL shader（如果可用）或 CSS filter 降级

### 首次加载

- 目标：首屏渲染 < 2s
- Desktop 场景的所有资源为首要加载项
- Archive 场景资源为次要加载（可在用户浏览 Desktop 时后台加载）
